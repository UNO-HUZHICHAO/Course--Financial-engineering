- **展示时长**：10-15 分钟

## 事件简介
2021年3月，英国供应链金融巨头 Greensill Capital 宣告破产。其商业模式将传统的“反向保理”变异为高风险的“未来应收账款”融资，并将这些无抵押信用贷款打包成证券化票据（Notes），销售给瑞信（Credit Suisse）的投资基金。同时，过度依赖单一保险公司（东京海上日动）提供信用违约保险。
当底层最大借款方（GFG集团）陷入危机，保险公司拒绝续保时，信用风险瞬间穿透至金融系统，导致瑞信被迫冻结并清算相关基金，造成百亿美元级的巨震。

---

## 整体分析框架 
1. **是什么 (What)**：Greensill 的表层业务与底层资产结构。
2. **为什么暴雷 (Why)**：集中度风险 + 信用增级失效 + 期限错配。
3. **怎么预测 (How/Quant)**：利用 KMV 模型计算瑞信在暴雷前夕的违约距离(DD)与预期违约概率(EDF)，证明市场数据对信用风险恶化的前瞻预警能力。
4. **怎么办 (Action)**：巴塞尔协议视角下的信用风控反思。

## PPT 任务拆解与排版指南 (给四位制作同学的 Tips)

### 模块一：案例背景与商业模式（同学 A 负责，约 3-4 页）
- **核心任务**：把复杂的供应链金融用人话讲清楚，说明 Greensill、瑞信、保险公司和借款企业的关系。
- **排版与视觉 Tips**：
  - **避坑**：千万不要堆砌大段文字介绍公司历史。
  - **必做图表**：画一张**“资金流与资产流转图”**。左边画实体企业（借钱），中间画 Greensill（打包），右边画瑞信和投资者（出钱），上面加个“保险伞”。用动画一步步点出这层嵌套关系。

### 模块二：风险识别与触发机制（同学 B 负责，约 3-4 页）
- **核心任务**：剖析风险点，解释资金链是怎么断裂的。
- **排版与视觉 Tips**：
  - **重点突出**：强调**“风险集中度”**。可以做一张饼图或柱状图，展示 Greensill 的资金有多大比例流向了 GFG 这一家公司。
  - **视觉引导**：在上一部分的“资金流转图”基础上，用显眼的红色 ❌ 打在“保险断裂”这个环节，直观展示信用违约是如何传染给瑞信的。

### 模块三：KMV量化建模展示（同学 C 负责，约 3 页）
- **核心任务**：配合建模成果，把高深的数学模型转化为清晰的学术图表。
- **排版与视觉 Tips**：
  - **避坑**：不要在 PPT 上放密密麻麻的偏微分方程和伊藤引理推导！台下会睡倒一片。
  - **图表展示**：留出大版面放置跑出来的 Python 可视化图表（违约距离走势图、EDF 激增图）。
  - **逻辑提炼**：用流程图（比如：数据输入 -> 提取市值/负债 -> 求解隐含资产 -> 输出违约概率）来代替复杂的公式说明。

### 模块四：案例启示与风控总结（同学 D 负责，约 2-3 页）
- **核心任务**：拔高立意，回到《金融风险管理》课程本身。
- **排版与视觉 Tips**：
  - **对比表格**：做一张表格，对比“Greensill 做法” vs “巴塞尔协议 III 信用风险管理要求”（如尽职调查穿透性、第三方担保的资本计提要求）。
  - **金句收尾**：总结提炼 3 条干货结论，排版要大气留白，为全场 Presentation 画上句号。
